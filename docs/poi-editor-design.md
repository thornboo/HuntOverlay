# POI 自定义编辑器 — 设计方案 (v1)

> 状态：设计稿，待用户在 Windows 验证重构基线后实现。
> 铁律对齐：纯本地数据编辑，不联网、不碰游戏进程/内存。

## 1. 现有数据结构（已核实）

`data.json` 是地图数组，每个地图：

```json
{
  "i": 1,
  "n": "Stillwater Bayou",
  "m": "./images/1.webp",
  "o": "./images/1.svg",
  "spawns":  [ {"c": [3877, 766]}, ... ],
  "armories":[ {"c": [x, y], "d": "描述", "u": ["图片URL"]}, ... ],
  ...
}
```

- 格式经 `detect_data_format` 判定为 **named**（因含 `n`）。
- 每个点位是 `{"c": [x, y]}`，可选 `d`(描述) / `u`(图片URL列表)。
- 坐标在 **0–4095** 网格；渲染时经 `rotate90cw_norm(x,y)` → 归一化 u,v。
- 渲染管线（overlay.py `build_for_category`）读 `it.get("c")`，跳过非 dict 或无 `c` 的项。

`poiData.json` 是每分类样式：`{type, label, categories, borderColor, fillColor, radius}`。

## 2. 核心约束：用户数据必须与远程数据分离

⚠️ **最关键的决策。** `data.json` 由 `fetch_remote_file` 从远程自动覆盖更新。
若把用户自定义点位写进 `data.json`，**下次更新会被冲掉**。

**方案：用户点位存独立文件 `user_pois.json`**，渲染时与 `data.json` 合并叠加。

```json
{
  "version": 1,
  "maps": {
    "Stillwater Bayou": {
      "armories": [ {"c": [1234, 5678], "d": "我的标注", "_user": true} ],
      "_custom_cat_xxx": [ ... ]
    }
  }
}
```

- 与 `data.json` 同 schema（`{"c":[x,y]}`），渲染管线可零改动复用。
- 每个用户点位打 `"_user": true` 标记，便于在叠加层区分显示/单独删除。
- 远程更新只动 `data.json`，`user_pois.json` 永不被覆盖。

## 3. 形态（建议 B：独立编辑器窗口）

从控制面板加「编辑点位」按钮打开独立 `PoiEditorDialog`。
- 比"叠加层内点击加点"(C) 简单可靠、易验证。
- 比"塞进现有面板"(A) 不挤占布局。

布局草图：
```
┌─ 编辑点位 ──────────────────────────┐
│ 地图: [静水河口 ▼]  分类: [军械库 ▼] │
│ ┌──────────────────────────────┐   │
│ │ # │  X   │  Y   │ 描述         │   │  ← 表格列出当前(用户+只读)点位
│ │ 1 │ 1234 │ 5678 │ 我的标注     │   │     只读点位灰显不可编辑
│ │ 2 │ ...                        │   │
│ └──────────────────────────────┘   │
│ [+ 新增点位] [删除选中] [保存] [取消] │
└──────────────────────────────────────┘
```

## 4. 第一版粒度（建议最小可用）

✅ 纳入：
- 选地图 + 分类，列出该分类下点位（远程只读 + 用户可编辑分开显示）
- 新增用户点位（输入 X/Y/描述，校验 0–4095）
- 删除用户点位
- 保存到 `user_pois.json`，触发叠加层刷新

❌ 暂不做（留 v2）：
- 自定义分类（用户自建一类）
- 叠加层内点击拾取坐标
- 编辑远程点位（远程只读，避免更新冲突）

## 5. 需要改动的位置（实现时）

| 模块 | 改动 | 风险 |
|------|------|------|
| `huntoverlay/user_data.py`(新) | 读写 `user_pois.json`、合并逻辑（纯函数，可单测） | 低 |
| `huntoverlay/widgets/poi_editor.py`(新) | 编辑器对话框 Qt UI | 中(Qt交互需真机验证) |
| `overlay.py` `build_for_category` | 合并 user_pois 点位 | 中(动渲染管线) |
| `widgets/panel.py` | 加「编辑点位」按钮 + 信号 | 低 |
| `runtime.py` | 加 `USER_POIS_PATH` | 低 |
| `tests/` | 合并逻辑/读写的单测 | 低 |

## 6. 安全/数据完整性检查清单

- [ ] 用户数据独立文件，远程更新永不覆盖
- [ ] 保存前校验坐标范围 0–4095
- [ ] 写入用 `save_json`（已有原子性可后续加强）
- [ ] 编辑器不发起任何网络请求
- [ ] 合并逻辑是纯函数，单元测试覆盖
